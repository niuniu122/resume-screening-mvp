# 模块日志：候选人姓名显示修复

## 时间
- 2026-03-27

## 问题描述
- 报告顶部出现“候选人：未识别”，用户希望直接显示姓名。
- 同时已有历史报告中还残留英文状态值，例如 `auto_reject_review`。

## 修复内容
- 文件：`backend/app/main.py`
  - 简历评估时，若正文未识别出姓名，优先将上传文件名（去后缀）作为候选人姓名提示传入评估引擎。
- 文件：`frontend/components/evaluation-drawer.tsx`
  - 新增姓名推断逻辑：优先显示 `evaluation.candidate_name`，否则退回到文件名。
  - 对历史 Markdown 报告做前端兼容替换：
    - `候选人：未识别` 替换为真实姓名/文件名
    - 当前状态替换为中文
    - `met / unmet / unknown` 替换为中文状态
- 文件：`backend/tests/test_workflow.py`
  - 新增测试，验证简历正文没有姓名时，会自动回退到文件名作为候选人姓名。

## 验证结果
- 后端测试：`pytest -q` 通过，共 `9 passed`
- 前端构建：`npm run build` 通过

## 用户侧效果
- 现在报告顶部会优先显示候选人姓名。
- 如果简历正文没写姓名，至少会显示上传文件名。
- 老报告重新打开时，也会自动把“未识别”和英文状态替换为中文可读内容。
